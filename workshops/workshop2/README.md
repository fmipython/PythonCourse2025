# Workshop 2 - Библиотека + приложения за метеорологични данни
*Преди да започнем, учтиво искаме да ви помолим да **не използвате** чат асистенти за цялостното решение на задачата.*

*Не възнаграждаваме с точки за успешно решение - целта е сами да се пробвате да помислите и напишете кода, за да има максимална полза за вас.*


## Какво?

Да се създаде **библиотека** на Python, която да предоставя две функционалности:
* проверка на текущо метеорологично време за дадена локация (име на град)
* дневна прогноза за времето за дадена локация (име на град) за няколко дни напред

Да се създаде **конзолно приложение** (CLI), което използва тази библиотека, за да предостави на потребителя гореизброените 2 функционалности.

Ако ви остане време, направете и допълнително **графично приложение** (GUI), което използва същата библиотека, за да демонстрира същите функционалности.


## Как?

1. За взимането на метеорологични данни използвайте някое от публичните API-та (евентуално и библиотеки), например:
   * https://wttr.in/ - не изисква ключ. Най-лесен и най-ограничен вариант. Една проста HTTP GET заявка към `https://wttr.in/{име_на_град}?format=j1` връща JSON с всички метеорологични данни, които ви вършат работа за задачата.
   * https://open-meteo.com/ - не изисква API ключ. Малко повече ориентиране из документацията трябва, но пък е стандартно REST API, специализирано точно за абсолютно всякакви видове метеоролигични данни и периоди. Имат Python SDK: https://github.com/open-meteo/python-requests
   * други REST APIs и/или библиотеки, които намерите
2. Напишете тестове:
   * юнит тестове за тези компоненти (функции или класове) на библиотеката ви, за които има смисъл
3. Напишете конзолното приложение, което използва библиотеката ви. Използвайте следните две библиотеки за улеснение:
    * [typer](https://typer.tiangolo.com/) - за лесно създаване на конзолни команди с модерен Pythonic стил
      * примерни команди:
        * `{име_на_скрипта.py} current Sofia` - показва текущото време в София
        * `{име_на_скрипта.py} daily Sofia` - показва дневната прогноза за времето в София за няколко дни напред
    * [rich](https://rich.readthedocs.io/en/stable/) - за красиво форматиране на изхода в конзолата (може например изходът да е под формата на [таблица](https://rich.readthedocs.io/en/stable/tables.html))
    * Ако не ви остане много време, може и да ползвате просто `sys.argv` и нормални `print()` изходи.
    * Ако някай желае да направи самата библиотека като конзолно приложение при директното ѝ изпълнение (т.е. с `python -m {име_на_пакета}`), може да го реализира чрез файл с име `__main__.py` в директорията на пакета и да имплементира там конзолната логика.
4. За графичното приложение (ако ви остане време) може да използвате например [streamlit](https://streamlit.io/) - с няколко реда Python код може да създадете уеб приложение, което например при въвеждане в текстово поле на град и кликване на бутон да показва текущото време и/или прогнозата за няколко дни напред.
